---

## What this package is

`centrcanon` is an open-source R package that provides the analytical engine for
**CEnTR\*MAP** — a Shiny-based workflow supporting institutions and communities in
collaboratively constructing a **local definition of community-engaged research
(CEnR)**.

### The end product is a definition, not just a map

The package does not produce a definition directly. It produces scores that help
practitioners identify which concepts are most central, structurally connected, and
influential in their institution's conceptual landscape. Those scores are then used —
by faculty, community partners, or working groups — to organize, prioritize, and
synthesize terms into a locally-grounded, community-constructed definition of CEnR.

In other words: the map is a means, not the end. The three pipelines surface the
underlying structure of how a community of scholars talks about their work, so that
structure can inform the language they ultimately adopt.

### The three pipelines

| Pipeline | Question answered | Output score |
|---|---|---|
| **Anchoring** | Which concepts are most central to how this community talks about CEnR? | `anchoring_score` |
| **Integration** | Which concepts serve as structural connectors across the network? | `integration_score` |
| **Leverage** | Which concepts hold the most influence in shaping the field? | `leverage_score` |

Together, high scores across all three pipelines identify the strongest candidates
for inclusion in a local CEnR definition. Clusters (from PAM) and communities (from
network detection) suggest how terms can be grouped into coherent themes or
commitments within that definition.

The Shiny application that calls this package is **closed-source and paywalled**.
The package itself must remain completely standalone — no Shiny imports, no UI code,
no app-specific configuration. All functions must work from the R console.

## Style guide

Follow the **[Tidyverse Style Guide](https://style.tidyverse.org/)** throughout.
Key rules:

- **snake_case** for all function names, argument names, and variables.
- **Informative verb-noun names**: `calculate_anchoring_score()`, `create_network()`,
  `classify_node_role()`. Avoid generic names like `process()` or `run()`.
- **One function per file** is a good default; group tightly related small helpers
  together (e.g., `utils-rank.R`).
- **No lines over 80 characters.** Wrap with the operator or pipe at the end of the
  preceding line.
- Use the **native pipe `|>`** (R ≥ 4.1), not `%>%`, unless a package-specific pipe
  is unavoidable.
- **No `library()` or `require()` calls** inside package functions. Use
  `package::function()` for non-imported calls, or declare in `NAMESPACE` via
  `@importFrom`.
- `@param`, `@return`, `@examples`, and `@export` roxygen tags are **required** for
  every exported function.
- Internal helpers get `@noRd` and **no** `@export`.
- Prefer `tibble` output over bare `data.frame` for all user-facing functions.
- Use `dplyr`/`tidyr` idioms for data manipulation; avoid base R `apply` loops unless
  performance demands it.
- Numeric output columns should be `double`; categorical outputs should be `factor`
  with explicit levels.

## Analytical pipelines — function contracts

### Shared utilities (`utils-rank.R`)

```r
# Min-max normalize a numeric vector; returns 0.5 for constant input
minmax_scale(x)  -> double vector in [0, 1]

# Shannon entropy normalized by log(n), where n = length(values)
calculate_entropy(values)  -> double scalar in [0, 1]

# Dense-rank then normalize to [0, 1]
dense_rank_normalize(x)  -> double vector in [0, 1]
```

### Tier classification (`tier.R`)

```r
calculate_tier(scores)
```
- Cuts `scores` at the 40th, 60th, and 80th percentiles.
- Returns an **ordered factor** with levels `"Low" < "Medium" < "High" < "Very High"`.
- Must handle `NA` values via `na.rm = TRUE` on internal `quantile()` calls.

### Pipeline 1 — Anchoring (`ca_analysis.R`, `pam_clustering.R`, `anchoring.R`)

**Step 0 — Prepare contingency table**

```r
prepare_contingency_table(edgelist)  -> matrix (groups × descriptors)
```
- Input: a tibble with columns `group`, `from`, `to` (the caller has already
  renamed raw source columns, e.g. `hex1`/`hex2`, to `from`/`to`).
- Pivots `from` and `to` into a single `descriptor` column so that both
  endpoints of each edge are treated as concept descriptors co-occurring with
  their group.
- Filters out `NA` and empty-string descriptors.
- Cross-tabulates `group × descriptor` via `table()`, then removes zero-sum
  rows and columns.
- Returns a matrix suitable for passing directly to
  `run_correspondence_analysis()`.

**Step 1 — Correspondence analysis**

```r
run_correspondence_analysis(contingency_table)  -> FactoMineR::CA object
```

**Step 2 — PAM clustering on CA coordinates**

```r
run_pam_clustering(ca_result, k = 6L)  -> cluster::pam object
```
- Default `k = 6L`; caller may override.
- Clusters on dimensions 1–2 of `ca_result$col$coord`.

**Step 3 — Anchoring score**

```r
calculate_anchoring_score(ca_result, cluster_assignments)
```

Returns a `tibble` with columns:

| Column | Type | Description |
|---|---|---|
| `concept` | character | concept / descriptor name |
| `cluster` | integer | PAM cluster id |
| `salience_raw` | double | CA column inertia |
| `dimensional_raw` | double | sum of contributions to dims 1 & 2 |
| `cluster_exemplar_raw` | double | proximity-weighted cos2 score |
| `rank_salience` | integer | dense rank of salience_raw |
| `rank_dimensional` | integer | dense rank of dimensional_raw |
| `rank_exemplar` | integer | dense rank of cluster_exemplar_raw |
| `combined_rank` | double | mean of the three ranks |
| `anchoring_score` | double | normalized to [0, 1] |
| `percentile_rank` | double | 0–100 |
| `anchoring_category` | ordered factor | "Peripheral" < "Supporting" < "Secondary" < "Primary" anchor |
| `tier` | ordered factor | from `calculate_tier()` |

The **exemplar sub-score** logic:
- For each cluster, compute the cluster centroid as `apply(coords, 2, median)`.
- Distance from centroid = Euclidean; invert and weight by `cos2_total`
  (sum of cos2 across dims 1 & 2).
- Single-member clusters get `cos2_total` directly as their exemplar score.

All three sub-scores are **equally weighted** (no boosting parameters in the
default exported function; leave a clearly documented argument stub if needed
for future extension).

### Pipeline 2 — Integration (`network.R`, `integration.R`)

**Network construction**

```r
create_network(edgelist)  -> igraph::igraph object
```
- Input: a two-column data frame / tibble with columns `from` and `to`
  (lowercase; document the rename from the source `From`/`To`).
- Calls `igraph::simplify()` to remove loops and multi-edges.
- Returns an **undirected** graph.

**Network metrics**

```r
calculate_network_metrics(graph)
```
Returns a `tibble` with one row per node and columns:

| Column | Source |
|---|---|
| `name` | `V(g)$name` |
| `degree` | `igraph::degree(g, normalized = TRUE)` |
| `betweenness` | `igraph::betweenness(g, normalized = TRUE)` |
| `crossclique` | `centiserve::crossclique(g)` |
| `cflow` | `centiserve::closeness.currentflow(g)` |
| `cbet` | `centiserve::communibet(g, normalized = TRUE)` |
| `eigen` | `igraph::eigen_centrality(g)$vector` |
| `katz` | `centiserve::katzcent(g)` |
| `community` | `igraph::cluster_optimal(g)$membership` as factor |

- `centiserve` functions should be wrapped in `tryCatch`; on failure, fill the
  column with `NA_real_` and emit a `rlang::warn()`.
- For graphs with `vcount(g) <= 1`, skip community detection and set
  `community = factor(1)`.

**Integration score**

```r
calculate_integration_score(network_metrics_tbl)
```
Returns the input tibble augmented with:

| Column | Logic |
|---|---|
| `betweenness_scaled` | min-max of `betweenness` |
| `cflow_scaled` | min-max of `cflow` |
| `crossclique_scaled` | min-max of `crossclique` |
| `rank_betweenness` | dense rank |
| `rank_cflow` | dense rank |
| `rank_crossclique` | dense rank |
| `rank_sum` | sum of three ranks |
| `integration_score` | normalized to [0, 1] |
| `tier` | `calculate_tier(integration_score)` |

### Pipeline 3 — Leverage (`leverage.R`)

**Leverage score + node classification**

```r
calculate_leverage_score(network_metrics_tbl)
```

Uses `eigen`, `katz`, and `cbet` columns (from `calculate_network_metrics()`).
Returns input tibble augmented with:

| Column | Logic |
|---|---|
| `eigen_z` | `minmax_scale(eigen)` |
| `katz_z` | `minmax_scale(katz)` |
| `cbet_z` | `minmax_scale(cbet)` |
| `rank_eigen` | dense rank of `eigen_z` |
| `rank_katz` | dense rank of `katz_z` |
| `rank_cbet` | dense rank of `cbet_z` |
| `rank_sum` | sum of three ranks |
| `leverage_score` | normalized to [0, 1] |
| `tier` | `calculate_tier(leverage_score)` |
| `node_role` | factor from `classify_node_role()` |

**Node role classification**

```r
classify_node_role(eigen_z, cbet_z, katz_z)  -> character scalar
```
Internal helper (`@noRd`). Classification logic (evaluated in order):

1. All three ≥ q90 → `"Core Keystone"`
2. All three ≥ q75 **and** entropy ≥ 0.9 → `"Supporting Keystone"`
3. All three ≥ q75 **and** entropy < 0.9 → paired type:
   - eigen + katz high → `"Beacon"`
   - eigen + cbet high → `"Steward"`
   - cbet + katz high → `"Aqueduct"`
   - else → `"Hybrid"`
4. All three ≤ q25 → `"Non-Key"`
5. Exactly one ≥ q75 → `"Sage"` (eigen), `"Weaver"` (cbet), `"Messenger"` (katz)
6. Two ≥ q75 → paired logic as in (3)
7. else → `"Leaning"`

Quantile thresholds (q25, q75, q90) must be computed **over all nodes in the
dataset being classified**, not hardcoded. Pass them as arguments to the internal
helper so tests can inject known values.

Node role levels (for the factor):
`"Core Keystone"`, `"Supporting Keystone"`, `"Beacon"`, `"Steward"`, `"Aqueduct"`,
`"Hybrid"`, `"Sage"`, `"Weaver"`, `"Messenger"`, `"Leaning"`, `"Non-Key"`

## Testing

Use **`testthat` (3rd edition)**. Declare `Config/testthat/edition: 3` in
`DESCRIPTION`.

Each test file mirrors its source file. Minimum coverage expectations:

- `test-tier.R`: boundary values, NA handling, correct factor levels and ordering.
- `test-anchoring.R`: `prepare_contingency_table()` produces correct dimensions
  and drops zero-sum rows/cols; score is in [0, 1]; single-concept cluster
  handled; factor levels correct; `calculate_tier()` output attached.
- `test-integration.R`: score is in [0, 1]; disconnected graph returns valid
  result; centiserve-absent path returns NA columns with a warning.
- `test-leverage.R`: each `classify_node_role()` branch reachable with a synthetic
  3-node case; `leverage_score` in [0, 1].

Build minimal synthetic fixtures in `tests/testthat/helper-fixtures.R` rather than
loading external CSV files.

## Data input conventions

The package does **not** own raw data loading — that belongs to the calling app.
The source CSV has columns `group`, `hex1`, `hex2`; the caller renames `hex1`/`hex2`
to `from`/`to` before passing the edgelist into any package function.

The single prepared input the package works from is:

| Input | Expected form |
|---|---|
| `edgelist` | Tibble with columns `group`, `from`, `to` (lowercase). An optional `weight` column is ignored by the CA pipeline but used by the network pipeline if present. |

From this edgelist, two analytical inputs are derived:
- **Contingency table** — via `prepare_contingency_table(edgelist)`, which the
  package provides. Both `from` and `to` are treated as descriptors co-occurring
  with their group; the result is a `group × descriptor` matrix.
- **Network graph** — via `create_network(edgelist)`, which treats `from`/`to`
  as directed node pairs and simplifies to an undirected graph.

Document this clearly in the vignette with a minimal worked example showing
the rename step and both downstream calls.

A small anonymized example dataset can live in `inst/extdata/` for use in tests
and the vignette. Cluster IDs returned by `run_pam_clustering()` are integers;
**human-readable cluster labels are the caller's responsibility** and must not be
hardcoded in the package.

## Build sequence

**Always build incrementally, phase by phase. Never write all files and run
`make check` in one shot — errors become hard to untangle.**

Each phase covers a coherent slice of the package. Complete each phase before
starting the next: write or edit the relevant `R/` file(s), then run the phase
target to document and test that slice.

| Phase | `make` target | R files | Test filter |
|---|---|---|---|
| 1 | `make phase1` | `utils-rank.R`, `tier.R` | `tier` |
| 2 | `make phase2` | `ca_analysis.R`, `pam_clustering.R`, `anchoring.R` | `anchoring` |
| 3 | `make phase3` | `network.R`, `integration.R` | `integration` |
| 4 | `make phase4` | `leverage.R` | `leverage` |
| 5 | `make phase5` | `palettes.R`, `vignettes/centr-pipeline.Rmd` | _(vignette build)_ |
| Final | `make check` | all | full `R CMD check` |

**Rules:**
- Run `make document` after every change to roxygen tags or exported function
  signatures, before running tests.
- Run the phase-specific test target (e.g. `make test-anchoring`) while
  developing a single file. Run `make test` only when a phase is complete.
- Do not proceed to the next phase until the current phase's tests pass and
  `make document` produces no warnings.
- Run `make check` only after all five phases are green. Resolve every NOTE,
  WARNING, and ERROR before committing.
- The one acceptable NOTE is `"New submission"` on a first CRAN submission.

**First-time setup:**
```bash
make setup      # install declared dependencies
make document   # generate NAMESPACE and man/
make phase1     # verify the foundation before writing anything else
```
